# MDX Authoring Guide

This page demonstrates the full MDX content pipeline available in Soroban-ZK-Std.

MDX lets you write standard **Markdown** and embed live **React components** in the same file — no extra imports needed for the built-in component set.

---

## What is MDX?

MDX is Markdown + JSX. You write prose in familiar Markdown syntax, and drop in React components wherever richer UI is needed.

<Alert type="info">
  All components listed in this guide are available in every `.mdx` file without any import statement.
</Alert>

---

```

```

## Text Formatting

Standard Markdown formatting works as expected:

- **Bold** — `**text**`
- *Italic* — `*text*`
- `Inline code` — `` `code` ``
- [Links](https://github.com/georgegoldman/Soroban-ZK-Std) — `[text](url)`

## Adding New MDX Pages

1. Create a `.mdx` file in `frontend/content/docs/`
2. Write standard Markdown with any built-in components
3. The page is automatically available at `/docs/content/<filename>`

### Adding New Components

1. Create the component in `frontend/components/mdx/`
2. Export it from `frontend/components/mdx/index.ts`
3. Register it in `frontend/mdx-components.tsx`
4. It becomes available in all `.mdx` files without imports
